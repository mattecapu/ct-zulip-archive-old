Fabrizio Genovese (Jan 16 2021 at 11:28):

Joe Moeller said:

I've also never understood what people don't like about Overleaf.

When you do a lot of coding, you start getting very picky wrt the editor you use. There are a lot of shortcuts and stuff you set up that make your work easier. For people using Emacs or Vim (not me) the level of customization is so high that it becomes practically impossible to give up on your macros without feeling a deep sense of frustration.

Fabrizio Genovese (Jan 16 2021 at 11:31):

This is similar to tiling window managers: As soon as you try them, get used to them and start defining your keyboard shortcuts getting back to a "normal" desktop environment become so frustrating that it's really difficult to describe how unnerving any single interaction with the new system becomes. It is exactly the same feeling I get in having to use overleaf editor. There are a lot of things that I don't like: Continuous compilation is the dumbest waste of computational power ever conceived, having to "live" in a browser tab makes me feel trapped and constrained in pointless ways, and overleaf macros feel like I'm using something akin to Microsoft Office, which says it all.

Fabrizio Genovese (Jan 16 2021 at 11:32):

As a general point: Forcing someone that has been working with computers for ages to change their to-go editor is a bad, bad idea. If you ever come up with something overleaf-like make sure to provide git integrations as they do!
