Nasos Evangelou-Oost (Apr 15 2021 at 12:33):

I'm trying to work with some labeled concrete categories, by which I mean for a concrete category CCC and a set LLL of labels, the labeled category is the comma category U/LU/LU/L where UUU is the forgetful C→SetC \to \mathsf{Set}C→Set and LLL is the constant functor on LLL.

My issue is that since the objects of these categories are set functions, when I do constructions like pullback, I just get a set function that I then need to manually show will lift to a structure-preserving map in CCC.

Is there something I can use to make sure the constructions on labelled objects automatically preserve/create the structure in CCC?

As an example: if CCC is the category of posets and f:X→Yf : X \to Yf:X→Y a set function and l:UP→Yl : UP \to Yl:UP→Y a YYY-labelled poset, then I would like to say that the pullback of f,lf, lf,l is an XXX-labeled poset but I'm not sure how.

Mike Shulman (Apr 23 2021 at 03:02):

However, if you really do mean posets and not preorders, then your original example:
Nasos Evangelou-Oost said:

if CCC is the category of posets and f:X→Yf : X \to Yf:X→Y a set function and l:UP→Yl : UP \to Yl:UP→Y a YYY-labelled poset, then I would like to say that the pullback of f,lf, lf,l is an XXX-labeled poset but I'm not sure how.

is false. For instance, let PPP be the terminal poset and lll the identity function; then the pullback is just XXX, and the only sensible preorder that can be induced on it is the indiscrete one, which is not antisymmetric.

Mike Shulman (Apr 23 2021 at 03:03):

And yes, for preorders you can use the right adjoint to the forgetful. Then the fact that it's a fibration just follows from the existence of pullbacks in Ord\mathsf{Ord}Ord.
